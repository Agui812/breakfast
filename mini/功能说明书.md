# 早餐店预约自取小程序 - 功能说明书

## 1. 项目概述

### 1.1 项目简介
本项目是一个面向早餐店的微信小程序，支持顾客在线点餐、预约自取、堂食选座，以及商家后台管理等功能。系统采用前后端分离架构，前端为微信小程序，后端提供 RESTful API 接口。

### 1.2 技术架构

| 层级 | 技术选型 |
|------|----------|
| 前端框架 | 微信小程序原生开发 |
| 后端接口 | RESTful API（Node.js） |
| 数据存储 | 后端数据库 + 本地缓存（localStorage） |
| 图片存储 | 本地文件系统 |

### 1.3 项目结构

```
mini/
├── pages/
│   ├── index/          # 点餐页面
│   ├── checkout/       # 结算页面
│   ├── order/          # 订单页面
│   ├── message/        # 消息页面
│   ├── mine/           # 我的页面
│   ├── login/          # 登录页面
│   ├── seat/           # 选座页面
│   ├── feedback/       # 反馈页面
│   └── admin/          # 管理后台
│       ├── index/      # 数据概览
│       ├── food/       # 餐品管理
│       └── order/      # 订单管理
├── utils/
│   ├── api.js          # API 请求封装
│   ├── storage.js      # 本地存储工具
│   └── util.js         # 通用工具函数
└── images/             # 静态资源
```

## 2. 用户端功能

### 2.1 用户登录

**功能描述**：支持微信授权登录和游客模式。

**功能要点**：
- 微信授权登录：获取用户昵称、头像，自动注册/登录
- 游客模式：可直接浏览餐品，但下单前需登录
- 登录状态持久化：用户信息保存在本地缓存

**数据字段**：
| 字段 | 类型 | 说明 |
|------|------|------|
| user_id | number | 用户ID |
| nickName | string | 用户昵称 |
| avatarUrl | string | 用户头像URL |

---

### 2.2 点餐页面

**功能描述**：展示餐品列表，支持分类筛选和购物车操作。

**功能要点**：
- 餐品展示：显示餐品名称、价格、库存、图片
- 分类筛选：按餐品分类筛选，支持"全部"分类
- 购物车操作：增加/减少餐品数量
- 购物车展开：点击底部购物车栏可展开查看已选餐品
- 实时计算：自动计算总数量和总价
- 库存控制：售罄餐品不可添加

**数据字段**：
| 字段 | 类型 | 说明 |
|------|------|------|
| id | number | 餐品ID |
| name | string | 餐品名称 |
| price | number | 单价 |
| dailyLimit | number | 每日限量 |
| stock | number | 当前库存 |
| status | string | 状态（onsale/soldout） |

---

### 2.3 结算页面

**功能描述**：确认订单信息，选择用餐方式和自取时间。

**功能要点**：
- 购物车确认：展示已选餐品清单
- 用餐方式选择：
  - 外带：直接下单
  - 堂食：跳转选座页面
- 自取时间选择：根据营业时间自动生成时间段（每30分钟一段）
- 联系电话：输入顾客手机号，支持保存历史号码
- 订单提交：调用后端 API 创建订单

**数据字段**：
| 字段 | 类型 | 说明 |
|------|------|------|
| phone | string | 顾客电话 |
| pickup_time | string | 自取时间 |
| dine_type | string | 用餐方式（takeout/dinein） |
| seat_number | string | 座位号（堂食时） |
| total_amount | number | 订单总金额 |
| user_id | number | 关联用户ID |

---

### 2.4 选座页面

**功能描述**：堂食顾客选择座位。

**功能要点**：
- 座位矩阵：4行6列，共24个座位（A1-F6）
- 座位状态：可选/已占用/已选中
- 互斥选择：同时只能选择一个座位
- 占用标记：选中的座位会被标记为占用
- 返回结算页：确认后返回结算页面继续下单

---

### 2.5 订单页面

**功能描述**：查看用户的历史订单和当前订单状态。

**功能要点**：
- 订单列表：展示当前用户的所有订单
- 订单状态：
  - 待制作（order_status = 1）
  - 已完成（order_status = 2）
  - 已取餐（order_status = 3）
- 订单详情：订单号、取餐码、自取时间、总金额
- 状态轮询：每10秒自动检查订单状态变化
- 取餐提醒：订单状态变为"已完成"时自动弹出 Toast 提醒
- 用户隔离：只显示当前用户（通过 user_id 或手机号匹配）的订单

---

### 2.6 消息页面

**功能描述**：集中展示用户的所有系统消息。

**功能要点**：
- 消息整合：统一展示下单成功、取餐提醒、商家回复
- 消息排序：按时间倒序排列
- 消息类型：
  - 下单成功：下单时自动生成
  - 取餐提醒：订单完成时自动生成
  - 商家回复：从反馈接口获取
- 用户隔离：只显示当前用户的消息
- 消息操作：点击"查看订单"跳转到订单页面

**消息数据结构**：
| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 消息ID |
| type | string | 消息类型（order/pickup/reply） |
| title | string | 消息标题 |
| desc | string | 消息内容 |
| time | string | 消息时间 |
| action | string | 操作类型 |
| actionText | string | 操作按钮文字 |
| unread | boolean | 是否未读 |

---

### 2.7 我的页面

**功能描述**：用户个人信息和快捷功能入口。

**功能要点**：
- 用户信息展示：头像、昵称
- 登录状态：已登录/未登录
- 营业时间展示：显示店铺营业开始和结束时间
- 快捷套餐预设：
  - 创建套餐：选择餐品、数量、自取时间、用餐方式
  - 编辑套餐：修改已有套餐
  - 删除套餐：删除不需要的套餐
  - 一键下单：点击套餐直接跳转结算页面
- 用户反馈：跳转到反馈页面

**套餐预设数据结构**：
| 字段 | 类型 | 说明 |
|------|------|------|
| name | string | 套餐名称 |
| foodList | array | 餐品列表 |
| items | string | 餐品描述文本 |
| totalPrice | number | 套餐总价 |
| timeSlot | string | 自取时间段 |
| dineType | string | 用餐方式 |

---

### 2.8 反馈页面

**功能描述**：用户提交使用反馈。

**功能要点**：
- 反馈类型选择：功能建议、使用体验、问题反馈、其他
- 反馈内容：输入反馈详情
- 联系方式：可选填写手机号或微信号
- 提交反馈：调用后端 API 提交

---

## 3. 管理后台功能

### 3.1 数据概览

**功能描述**：展示今日经营数据。

**功能要点**：
- 今日订单数：统计今日订单总量
- 今日营收：统计今日订单总金额
- 今日餐品数：统计今日售出餐品总数
- 快捷入口：跳转到餐品管理、订单管理

---

### 3.2 餐品管理

**功能描述**：管理餐品信息。

**功能要点**：
- 餐品列表：展示所有餐品
- 添加餐品：输入名称、价格、每日限量
- 编辑餐品：修改餐品信息
- 删除餐品：删除餐品（需确认）
- 库存管理：自动扣减库存

---

### 3.3 订单管理

**功能描述**：管理顾客订单。

**功能要点**：
- 订单列表：展示所有订单
- 订单筛选：按状态筛选（全部/待制作/已完成/已取餐）
- 完成订单：标记订单为"已完成"，触发取餐提醒
- 核销订单：标记订单为"已取餐"
- 打印小票：展示订单详情供打印
- 回复顾客：通过反馈接口回复顾客

**订单操作 API**：
| 操作 | API | 说明 |
|------|-----|------|
| 完成订单 | PUT /api/orders/:id/status | order_status = 2 |
| 核销订单 | PUT /api/orders/:id/status | order_status = 3 |
| 回复顾客 | PUT /api/feedbacks/:id/reply | 更新反馈回复 |

---

## 4. 消息通知机制

### 4.1 消息生成规则

| 消息类型 | 触发条件 | 生成方式 |
|----------|----------|----------|
| 下单成功 | 用户提交订单 | 订单创建后自动生成 |
| 取餐提醒 | 订单状态变为"已完成" | 订单页面轮询检测 |
| 商家回复 | 商家回复用户反馈 | 从反馈接口获取 |

### 4.2 取餐提醒轮询机制

- 轮询间隔：10秒
- 检测逻辑：对比订单状态变化
- 触发条件：order_status 从 1（待制作）变为 2（已完成）
- 提醒方式：Toast 弹窗 + 消息列表新增记录
- 生命周期：页面显示时启动，隐藏/卸载时停止

### 4.3 用户隔离机制

- 订单匹配：通过 `user_id` 或 `phone` 匹配当前用户
- 消息过滤：只显示当前用户的订单消息和反馈回复
- 数据同步：所有数据从后端 API 获取，保证一致性

### 7.2 页面跳转关系

```

  └── 结算页面 (checkout)
        ├── 选座页面 (seat) → 返回结算页面

  └── 消息页面 (message) [TabBar]

我的页面 (mine)
  ├── 登录页面 (login)

  └── 结算页面 (checkout) [快捷套餐下单]

管理后台
  ├── 数据概览 (admin/index)
  ├── 餐品管理 (admin/food)
  └── 订单管理 (admin/order)
```

## 8. 营业时间配置

### 8.1 配置说明

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 营业开始时间 | 06:00:00 | 店铺开门时间 |
| 营业结束时间 | 10:00:00 | 店铺关门时间 |

### 8.2 时间段生成规则

- 根据营业时间自动生成时间段
- 每30分钟一个时间段
- 示例：06:00-06:30、06:30-07:00、07:00-07:30...

## 9. 订单状态流转

```
待制作 (1) ─→ 已完成 (2) ─→ 已取餐 (3)
   │              │
   │              ── 触发取餐提醒
   ── 商家可操作完成
```

| 状态 | 值 | 说明 |
|------|-----|------|
| 待制作 | 1 | 订单已提交，等待制作 |
| 已完成 | 2 | 餐品制作完成，可取餐 |
| 已取餐 | 3 | 顾客已取餐，订单完成 |

## 10. 安全与权限

### 10.1 用户认证

- 微信授权登录：通过 openid 识别用户
- 游客模式：可浏览但不可下单
- 登录状态持久化：本地缓存用户信息

### 10.2 数据隔离

- 订单隔离：用户只能查看自己的订单
- 消息隔离：用户只能查看自己的消息
- 匹配规则：user_id 优先，手机号作为备选

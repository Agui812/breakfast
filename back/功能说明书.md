# 早餐店预约自取系统 - 功能说明书

## 一、项目概述

### 1.1 系统简介
早餐店预约自取系统是一套面向早餐店的数字化管理解决方案，包含微信小程序端（顾客使用）和后台管理系统（商家使用）。系统支持顾客在线预约下单、商家后台管理餐品和订单、自动打印订单等功能。

### 1.2 技术架构

| 层级 | 技术栈 |
|------|--------|
| 前端框架 | Vue 3 + Vite + Element Plus |
| 后端框架 | Node.js + Express |
| 数据库 | MySQL 8.0 |
| 部署方式 | 本地部署 |

### 1.3 系统地址

| 服务 | 地址 |
|------|------|
| 后端 API | http://localhost:3001 |
| 前端管理后台 | http://localhost:5173 |

---

## 二、功能模块

### 2.1 数据概览（首页）

首页提供经营数据的可视化展示，帮助商家快速了解店铺运营状况。

| 功能 | 说明 |
|------|------|
| 餐品总数 | 显示当前在售餐品数量 |
| 订单总数 | 显示历史累计订单数量 |
| 今日订单 | 显示当日新增订单数量 |
| 今日营收 | 显示当日营业收入金额 |
| 待处理订单 | 显示待制作订单数量，点击可跳转至订单列表 |
| 待处理反馈 | 显示待处理用户反馈数量，点击可跳转至反馈列表 |

### 2.2 用户管理

管理通过微信小程序注册的用户信息。

| 功能 | 说明 |
|------|------|
| 用户列表 | 查看所有注册用户，显示头像、昵称、OpenID |
| 编辑用户 | 修改用户昵称和头像 |
| 查看订单 | 查看指定用户的历史订单记录 |

### 2.3 分类管理

管理餐品分类，方便餐品的组织和筛选。

| 功能 | 说明 |
|------|------|
| 分类列表 | 查看所有餐品分类 |
| 添加分类 | 新增餐品分类，设置名称、排序、状态 |
| 编辑分类 | 修改分类信息 |
| 删除分类 | 删除不需要的分类 |
| 启用/禁用 | 控制分类的显示状态 |

### 2.4 餐品管理

管理店铺销售的餐品信息，包括价格、库存、图片等。

| 功能 | 说明 |
|------|------|
| 餐品列表 | 查看所有餐品，支持按分类筛选 |
| 添加餐品 | 新增餐品，设置分类、名称、价格、库存、图片 |
| 编辑餐品 | 修改餐品信息 |
| 删除餐品 | 下架餐品 |
| 图片上传 | 支持本地图片上传，自动保存到服务器 |
| 库存管理 | 设置每日限量和当前剩余库存 |
| 状态管理 | 控制餐品在售/售罄状态 |

### 2.5 订单管理

管理顾客提交的预约订单。

| 功能 | 说明 |
|------|------|
| 订单列表 | 查看所有订单，支持按状态、日期筛选 |
| 订单详情 | 查看订单完整信息，包括订单明细 |
| 状态更新 | 更新订单状态（待制作→已完成→已取餐） |
| 自动打印 | 创建订单时自动记录打印日志 |
| 分页查询 | 支持分页浏览大量订单 |

**订单状态说明**：

| 状态 | 说明 |
|------|------|
| 待制作 | 订单已提交，等待商家制作 |
| 已完成 | 餐品已制作完成，等待顾客取餐 |
| 已取餐 | 顾客已取走餐品，订单完成 |

### 2.6 用户反馈

管理顾客提交的反馈和建议。

| 功能 | 说明 |
|------|------|
| 反馈列表 | 查看所有用户反馈，支持按状态筛选 |
| 回复反馈 | 商家回复用户反馈，自动标记为已处理 |
| 删除反馈 | 删除无效或已处理的反馈 |
| 状态管理 | 标记反馈为待处理/已处理 |

### 2.7 打印日志

记录所有订单的打印历史。

| 功能 | 说明 |
|------|------|
| 日志列表 | 查看所有打印记录 |
| 订单明细 | 每条打印记录包含订单明细信息 |
| 分页查询 | 支持分页浏览 |

### 2.8 系统配置

配置店铺营业时间和其他系统参数。

| 功能 | 说明 |
|------|------|
| 营业时间设置 | 设置营业开始和结束时间 |
| 配置更新 | 实时生效，影响小程序端的可预约时间 |

## 三、数据库设计

### 3.1 数据表清单

| 表名 | 说明 | 主要字段 |
|------|------|----------|
| user | 用户表 | id, openid, nickname, avatar |
| category | 餐品分类表 | id, name, sort, status |
| product | 餐品表 | id, category_id, name, price, daily_stock, current_stock, image, status |
| order_info | 订单主表 | id, order_no, pickup_code, nickname, phone, user_id, pickup_time, total_amount, order_status |
| order_item | 订单明细表 | id, order_no, product_id, product_name, price, quantity |
| system_config | 系统配置表 | id, business_start, business_end |
| print_log | 打印日志表 | id, order_no, print_time |
| feedback | 用户反馈表 | id, user_id, content, contact, status, reply |

### 3.2 核心业务流程

**下单流程**：
1. 顾客在小程序选择餐品和取餐时间
2. 提交订单，系统生成订单编号和取餐码
3. 系统自动扣减对应餐品库存
4. 系统自动记录打印日志
5. 商家在后台查看订单并制作
6. 顾客凭取餐码到店取餐

## 五、部署说明

### 5.1 环境要求

| 组件 | 版本要求 |
|------|----------|
| Node.js | >= 16.0 |
| MySQL | >= 8.0 |
| npm | >= 8.0 |

### 5.2 后端启动

```bash
cd back
npm install
npm start
```

### 5.3 前端启动

```bash
cd back/admin
npm install
npm run dev
```

### 5.4 数据库初始化

```bash
mysql -u root -p < sql.sql
```

## 六、注意事项

1. 图片上传功能需要确保 `public/uploads` 目录存在且有写入权限
2. 订单创建时会自动扣减库存，请注意库存管理
3. 打印日志在订单创建时自动记录，无需手动操作
4. 系统配置修改后实时生效，影响小程序端的可预约时间段
5. 用户反馈回复后会自动标记为已处理状态

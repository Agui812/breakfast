# 早餐店预约自取系统

面向早餐店的数字化管理解决方案，支持顾客在线预约下单、到店即取，解决早高峰排队痛点。系统包含微信小程序端（顾客使用）和后台管理系统（商家使用）。

## 技术架构

| 层级 | 技术栈 |
|------|--------|
| 小程序端 | 微信小程序原生开发 |
| 管理后台前端 | Vue 3 + Vite + Element Plus + Pinia |
| 后端服务 | Node.js + Express |
| 数据库 | MySQL 8.0 |

## 功能模块

### 顾客端（微信小程序）

- **用户登录**：微信授权登录 + 游客模式，登录状态本地持久化
- **点餐**：餐品列表浏览、分类筛选、购物车操作、实时库存展示
- **结算**：选择用餐方式（外带/堂食）、自取时间段、填写联系电话
- **堂食选座**：堂食模式下选择座位
- **订单管理**：查看历史订单、取餐码、订单状态跟踪
- **消息中心**：接收订单状态变更通知
- **用户反馈**：提交反馈建议和联系方式
- **小程序管理后台**：数据概览、餐品管理、订单管理（管理员入口）

### 商家端（后台管理系统）

- **数据概览**：餐品总数、订单总数、今日订单、今日营收、待处理订单/反馈
- **用户管理**：查看微信注册用户、编辑信息、查看用户订单
- **分类管理**：餐品分类的增删改查、启用/禁用
- **餐品管理**：餐品增删改查、图片上传、每日限量/库存管理、在售/售罄状态
- **订单管理**：订单列表（按状态/日期筛选）、状态流转（待制作 → 已完成 → 已取餐）
- **打印日志**：订单打印记录
- **用户反馈**：查看/处理/回复用户反馈
- **系统配置**：设置营业时间段

### 订单状态流转

```
待制作 → 已完成 → 已取餐
```

## 数据库设计

共 8 张表，建表脚本位于 `back/sql.sql`：

| 表名 | 说明 |
|------|------|
| `user` | 微信用户（openid、昵称、头像） |
| `category` | 餐品分类（名称、排序、状态） |
| `product` | 餐品（名称、价格、每日限量、当前库存、图片） |
| `order_info` | 订单主表（订单号、取餐码、自取时间、总金额、状态） |
| `order_item` | 订单明细（餐品快照、单价、数量） |
| `system_config` | 系统配置（营业开始/结束时间） |
| `print_log` | 打印日志（订单号、打印时间） |
| `feedback` | 用户反馈（内容、联系方式、处理状态、商家回复） |

## 快速开始

### 1. 数据库初始化

```bash
# 登录 MySQL 执行建表脚本
mysql -u root -p < back/sql.sql
```

### 2. 启动后端服务

```bash
cd back
npm install
npm run dev      # 开发模式（nodemon 热重载）
# 或
npm start        # 生产模式
```

后端服务运行在 `http://localhost:3001`

### 3. 启动管理后台前端（开发模式）

```bash
cd back/admin
npm install
npm run dev
```

前端开发服务运行在 `http://localhost:5173`

> 生产环境直接使用 `npm start` 启动后端，Express 会自动托管 `admin/dist` 下的构建产物，访问 `http://localhost:3001` 即可。

### 4. 小程序端

使用微信开发者工具打开 `mini/` 目录，配置 AppID 后即可运行。

## API 接口

后端提供 RESTful API，基础路径为 `/api`，主要模块：

| 路径 | 说明 |
|------|------|
| `/api/auth` | 管理员认证 |
| `/api/users` | 用户管理（含微信小程序登录） |
| `/api/products` | 餐品管理 |
| `/api/categories` | 分类管理 |
| `/api/orders` | 订单管理 |
| `/api/config` | 系统配置 |
| `/api/dashboard` | 数据概览 |
| `/api/upload` | 图片上传 |
| `/api/feedbacks` | 用户反馈 |
| `/api/print-logs` | 打印日志 |

详细接口文档参见 `back/API接口文档.md` 和 `mini/API接口文档.md`。
